# S03 – eda_cli: мини-EDA для CSV

Небольшое CLI-приложение для базового анализа CSV-файлов.  
Используется в рамках Семинара 03 курса «Инженерия ИИ».

## Требования

- Python 3.11+
- [uv](https://docs.astral.sh/uv/) установлен в систему

## Инициализация проекта

В корне проекта (S03):

```bash
uv sync
```

Эта команда:

- создаёт виртуальное окружение `.venv`;
- устанавливает зависимости из `pyproject.toml`;
- устанавливает сам проект `eda-cli` в окружение.

## Запуск CLI

В проекте доступны следующие команды командной строки.

---

## Команда `overview`

Команда выводит краткий обзор датасета:

- количество строк и столбцов;
- типы данных;
- сводную таблицу по колонкам.

Пример запуска:

```bash
uv run eda-cli overview data/example.csv
```

Доступные параметры:

- `--sep` — разделитель в CSV-файле (по умолчанию `,`);
- `--encoding` — кодировка файла (по умолчанию `utf-8`).

---

## Команда `report`

Команда генерирует полный EDA-отчёт по CSV-файлу.

Пример базового запуска:

```bash
uv run eda-cli report data/example.csv --out-dir reports
```

В результате в каталоге `reports/` создаются следующие файлы и папки:

- `report.md` — основной отчёт в формате Markdown;
- `summary.csv` — таблица со сводной информацией по колонкам;
- `missing.csv` — статистика пропусков по колонкам;
- `correlation.csv` — корреляционная матрица (если есть числовые признаки);
- `top_categories/*.csv` — таблицы top-k значений для категориальных признаков;
- `hist_*.png` — гистограммы числовых колонок;
- `missing_matrix.png` — визуализация пропусков;
- `correlation_heatmap.png` — тепловая карта корреляций.

---

## Дополнительные параметры команды `report`

Команда `report` поддерживает набор параметров, влияющих на содержимое отчёта.

### Параметры визуализации и анализа

- `--max-hist-columns`  
  Максимальное количество числовых колонок, для которых строятся гистограммы  
  (по умолчанию: `6`).

- `--max-cat-columns`  
  Максимальное количество категориальных колонок, для которых считается статистика top-значений  
  (по умолчанию: `5`).

- `--top-k-categories`  
  Количество top-значений, выводимых для каждого категориального признака  
  (по умолчанию: `5`).

### Параметры качества данных и отчёта

- `--min-missing-share`  
  Порог доли пропусков (в диапазоне `0..1`).  
  Колонки с `missing_share >= порога` считаются проблемными и выводятся отдельным списком в отчёте  
  (по умолчанию: `0.3`).

- `--title`  
  Заголовок отчёта — первая строка вида `# ...` в файле `report.md`.

Все значения указанных параметров фиксируются в разделе «Настройки отчёта» внутри `report.md`.

---

## Пример вызова `report` с новыми параметрами

```bash
uv run eda-cli report data/example.csv   --out-dir reports_run   --title "EDA report for example.csv"   --max-hist-columns 8   --max-cat-columns 4   --top-k-categories 7   --min-missing-share 0.15
```
